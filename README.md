# طبيب عن بعد: الواجهة الخلفية (Backend)

**منصة استجابة طبية سريعة، مصممة للعمل في أصعب الظروف.**

هذا المستودع يحتوي على الكود المصدري للواجهة الخلفية (Backend) لمشروع "طبيب عن بعد". المشروع يهدف إلى بناء جسر فوري بين الحالات الطارئة في المناطق ذات الموارد المحدودة وشبكة عالمية من الأطباء المتطوعين.

---

## 🚀 الميزات الأساسية

- **بوابة الطوارئ:** وصول فوري للمساعدة عبر رقم الهاتف فقط، بدون تعقيدات إنشاء حساب، لضمان أسرع استجابة ممكنة.
- **تعيين ذكي للأطباء:** نظام تلقائي يوزع الحالات على الطبيب المتطوع الأنسب بناءً على التخصص وعبء العمل الحالي.
- **إشعارات فورية حية:** باستخدام Django Channels و WebSockets، يتم إشعار الأطباء بالحالات الجديدة في نفس اللحظة.
- **جاهزية للعمل دون اتصال:** تم تصميم الواجهات الخلفية لدعم سيناريو انقطاع الإنترنت، عبر استخدام ID فريد لكل حالة يتم إنشاؤها على الجهاز.
- **معالجة الصور في الخلفية:** يتم ضغط الصور المرفوعة في مهمة خلفية (`django-background-tasks`) لضمان سرعة الاستجابة وعدم انتظار المستخدم، وهو أمر حاسم في الشبكات الضعيفة.
- **توثيق تلقائي للـ API:** يتم إنشاء توثيق تفاعلي وواضح لكل الواجهات البرمجية باستخدام `drf-spectacular`، مما يسهل التعاون مع فريق الواجهة الأمامية.
- **جودة وموثوقية عالية:** المشروع مدعوم بحصن من **21 اختباراً آلياً** يغطي كل الوظائف الحرجة لضمان استقرار النظام.

---

## 🛠️ البنية التقنية

- **الإطار (Framework):** Django, Django REST Framework
- **الاتصالات الفورية:** Django Channels
- **المهام الخلفية:** Django Background Tasks
- **المصادقة:** Simple JWT (JSON Web Tokens)
- **قاعدة البيانات:** جاهز للعمل على PostgreSQL (يستخدم SQLite للتطوير)
- **التوثيق:** drf-spectacular (Swagger UI / ReDoc)

## ⚙️ الإعداد والتشغيل محلياً

لتشغيل المشروع على جهازك المحلي، اتبع الخطوات التالية:

1.  **استنساخ المستودع:**
    ```bash
    git clone https://github.com/your-username/remote-doctor-backend.git
    cd remote-doctor-backend
    ```

2.  **إنشاء وتفعيل البيئة الافتراضية:**
    ```bash
    # Windows
    python -m venv venv
    venv\Scripts\activate
    ```

3.  **تثبيت الاعتماديات:**
    ```bash
    pip install -r requirements.txt
    ```

4.  **إعداد متغيرات البيئة:**
    - قم بإنشاء ملف `.env` في المجلد الرئيسي.
    - انسخ محتوى ملف `.env.example` (إذا وجد) أو استخدم القالب التالي:
      ```
      SECRET_KEY=your-secret-key
      DEBUG=True
      ALLOWED_HOSTS=127.0.0.1,localhost
      DATABASE_URL=sqlite:///db.sqlite3
      ```

5.  **تطبيق الترحيلات (Migrations):**
    ```bash
    python manage.py migrate
    ```

6.  **تشغيل الخوادم (في طرفيتين منفصلتين):**
    - **الطرفية الأولى (الخادم الرئيسي):**
      ```bash
      python -m daphne core.asgi:application
      ```
    - **الطرفية الثانية (معالج المهام):**
      ```bash
      python manage.py process_tasks
      ```

الآن، يجب أن يكون المشروع يعمل على `http://127.0.0.1:8000/`.
